# 实时会议转录验证系统

本项目包含一个 Python FastAPI 后端服务及一个单页面的 HTML 前端应用。Python 端作为主服务中枢，既承担访问页面的 HTTP 宿主，也负责承担转发前端降采样后捕获的 WebSocket 录音，最后对接给您真实部署在内网中的 `FunASR Docker` 转录服务器。

## 环境准备
请确保您当前计算机已安装有 Python 环境。
在项目目录下打开终端（Terminal 或 CMD 命令行），执行以下命令安装运行所需的模块库：

```powershell
pip install -r requirements.txt
```
*(如果网络缓慢或卡顿，推荐添加国内镜像源 `-i https://pypi.tuna.tsinghua.edu.cn/simple`)*

## 注意配置真实的转录地址
根据之前的约定，如果没有使用默认的本地测试地址 `ws://127.0.0.1:10095`，请先通过编辑器**打开 `main.py`** ：
找到第 20 行：
```python
FUNASR_WS_URL = os.getenv("FUNASR_WS_URL", "ws://127.0.0.1:10095")
```
修改为您公司内网 FunASR 服务真实的 WebSocket 监听地址。

---

## ▶ 如何启动服务应用（后端）

本应用将后端的 WebSocket 流对接与前台的静态网页分发集合为了一个主进程服务，所以启动这一行命令即可拉起所有逻辑。

在终端内运行：
```powershell
uvicorn main:app --host 0.0.0.0 --port 8000
```

> 提示：当在控制台看到类似于 `Application startup complete.` 和 `启动 FastAPI 应用服务...` 时，表明接口和服务准备完毕。

---

## 🌐 如何访问与测试（前端）

1. **访问页面**：服务启动后，直接使用 Chrome / Edge 等现代浏览器访问：
   ▶ `http://127.0.0.1:8000/`

2. **开始转录**：
   - 网页加载完成后，点击页面上方的**【开始录音】**按钮。
   - 浏览器将会在顶部提示需要使用麦克风权限，**请点击【允许】**。
   - 这时您对着麦克风正常讲话，您的浏览器后台会将您麦克风声音进行下采样切分为 `16kHz PCM` 后不断传递给 Python 后端。

3. **结果观察**：
   - 如果内网 `FunASR` 服务通畅，转录文字会很快实时在页面下方的白色方块区域刷出，实时变动的部分会显现出灰色，确认为最终完整的一句会变为黑色加粗体。
   - 与此同时，您也会在启动后端的那个命令行后台里，实时看到所有包含`[转录完成]`或`[实时]`的企业级中文调试记录。

4. **遇到网络波动**：
   如果您故意在测试中断掉 `FunASR` Docker 并马上恢复，你会能够在控制台看到 `断线异常，2秒后自动退避重连` 的日志，在此期间网页连接不会中断并将在接通后重新转发。
